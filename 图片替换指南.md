# 图片替换指南

本文档说明如何替换网站中的产品图片和 Banner 图片。

## 目录结构

```
public/
├── images/
│   ├── products/          # 产品图片
│   │   └── [分类文件夹]/
│   │       ├── [产品slug].jpg
│   │       └── [产品slug].png
│   └── banners/          # Banner 轮播图
│       ├── 1.jpg
│       ├── 2.jpg
│       └── 3.jpg
```

## 产品图片替换

### 1. 按分类组织图片

在 `public/images/products/` 下创建对应的分类文件夹：

```
public/images/products/
├── pm-patch-cord/          # 保偏跳线
├── high-density/           # 高密度连接组件
├── test-solutions/        # 测试方案
└── specialty/             # 特种线缆
```

### 2. 命名规范

产品图片文件名应与产品文件的 `slug` 一致：

- **产品文件**: `src/content/products/zh/pm-patch-cord-lc-apc-1m.md`
- **图片文件**: `public/images/products/pm-patch-cord/pm-patch-cord-lc-apc-1m.jpg`

### 3. 图片格式

- **推荐格式**: JPG（照片类）、PNG（带透明背景）
- **尺寸建议**:
  - 产品主图: 800×600 或 1200×900 像素
  - 缩略图: 300×225 像素
- **文件大小**: 每张图片建议 < 500KB

### 4. 在产品文件中引用

在产品 Markdown 文件中设置 `image` 字段：

```yaml
---
title: "保偏跳线 LC/APC 1M"
slug: "pm-patch-cord-lc-apc-1m"
category: "pm-patch-cord"
image: "/images/products/pm-patch-cord/pm-patch-cord-lc-apc-1m.jpg"
---
```

**注意**: 路径必须以 `/` 开头，相对于 `public/` 目录。

## Banner 图片替换

### 1. Banner 位置

Banner 图片存储在 `public/images/banners/` 目录。

### 2. 当前 Banner 列表

```
public/images/banners/
├── 1.jpg    # 主 Banner（光纤跳线产品）
├── 2.jpg    # 次 Banner（高密度连接）
└── 3.jpg    # 次 Banner（测试方案）
```

### 3. Banner 尺寸建议

- **推荐尺寸**: 1920×600 像素
- **最小尺寸**: 1600×500 像素
- **比例**: 约 3.2:1
- **格式**: JPG（推荐）、PNG
- **文件大小**: 建议 < 1MB

### 4. 通过 CMS 管理 Banner

你也可以通过 CMS 后台管理 Banner：

1. 访问: http://localhost:4321/admin/
2. 点击 "Banners"
3. 点击 "Add new banner"
4. 填写信息:
   - **Title**: Banner 标题
   - **Image**: 上传或输入图片路径
   - **Alt**: 图片描述（SEO）
   - **Button Text**: 按钮文字
   - **Button Link**: 按钮链接
   - **Order**: 显示顺序（数字越小越靠前）

## 图片优化建议

### 1. 压缩图片

使用在线工具压缩图片，提升加载速度：
- **TinyPNG**: https://tinypng.com/
- **Squoosh**: https://squoosh.app/
- **ImageOptim**: macOS 本地工具

### 2. 使用 WebP 格式（可选）

WebP 格式文件更小，质量更高：

```bash
# 使用 cwebp 工具转换
cwebp input.jpg -o output.webp -q 80
```

如果使用 WebP，记得更新文件引用：

```yaml
image: "/images/products/pm-patch-cord/pm-patch-cord-lc-apc-1m.webp"
```

### 3. 多尺寸支持（高级）

为不同设备提供不同尺寸：

```
public/images/products/pm-patch-cord/
├── pm-patch-cord-lc-apc-1m-320.jpg    # 移动端
├── pm-patch-cord-lc-apc-1m-640.jpg    # 平板
└── pm-patch-cord-lc-apc-1m-1200.jpg   # 桌面
```

在产品文件中指定：

```yaml
image: "/images/products/pm-patch-cord/pm-patch-cord-lc-apc-1m-1200.jpg"
sizes: [320, 640, 1200]
```

## 常见问题

### Q: 图片上传后不显示？

**A**: 检查以下几点：
1. 文件路径是否正确（必须以 `/` 开头）
2. 文件名大小写是否匹配（Linux 区分大小写）
3. 文件是否在 `public/` 目录下
4. 开发服务器是否重启（有时候需要）

### Q: 如何批量替换产品图片？

**A**: 使用脚本批量处理：

```bash
# Windows PowerShell
# 将所有图片从源目录复制到目标目录，保持文件名一致
Copy-Item -Path "C:\来源目录\*.jpg" -Destination "d:\360MoveData\Users\nikey\Desktop\光纤跳线网站\public\images\products\分类\"
```

### Q: Banner 图片不轮播？

**A**: 检查 CMS 中的 Banner 配置：
1. 是否有多个 Banner 启用（`active: true`）
2. `order` 字段是否设置正确
3. Banner 图片是否存在

### Q: 图片上传到 CMS 后路径是什么？

**A**:
1. CMS 不支持直接上传图片文件到 `public/`
2. 需要手动将图片放入 `public/images/` 对应目录
3. 然后在 CMS 中输入图片路径（如 `/images/products/xxx.jpg`）

## 图片来源建议

### 产品图片
- **实际产品照片**: 使用专业相机拍摄白底产品图
- **渲染图**: 使用 3D 软件渲染高质量产品图
- **供应商图片**: 确保有使用授权

### Banner 图片
- **图库网站**: Unsplash、Pexels、Pixabay（免费商用）
- **自行设计**: Photoshop、Figma、Canva
- **行业素材**: 光纤、科技、工业风格图片
