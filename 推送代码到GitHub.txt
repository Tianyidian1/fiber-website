# 代码推送到 GitHub

## 前置条件

1. ✅ Git 已安装并配置
2. ✅ 本地仓库已初始化并提交
3. ⚠️ 需要创建 GitHub Personal Access Token

---

## 第一步：创建 GitHub Personal Access Token

1. 登录 GitHub：https://github.com
2. 点击右上角头像 → **Settings**
3. 左侧菜单最下方 → **Developer settings**
4. 左侧 → **Personal access tokens** → **Tokens (classic)**
5. 点击 **Generate new token (classic)**
6. 填写：
   - **Note**：fiber-website 部署
   - **Expiration**：选择 90 days 或 No expiration
   - **Scopes**：勾选以下权限：
     - ✅ `repo`（完整仓库权限）
     - ✅ `workflow`（GitHub Actions 权限）
7. 点击 **Generate token**
8. **重要**：复制生成的 token（以 `ghp_` 开头），**只显示一次！**

---

## 常见问题

### 问题 1：提示 "Authentication failed"

**原因**：Token 输入错误或已过期

**解决**：
1. 删除本地保存的凭据
2. 重新生成 Token
3. 再次推送

### 问题 2：提示 "remote origin already exists"

**原因**：之前已经关联过远程仓库

**解决**：
```bash
cd "d:\360MoveData\Users\nikey\Desktop\光纤跳线网站"

# 查看当前远程仓库
git remote -v

# 如果不对，删除后重新添加
git remote remove origin
git remote add origin https://github.com/Tianyidian1/fiber-website.git

# 推送
git push -u origin main
```

### 问题 3：提示 "Updates were rejected"

**原因**：远程仓库已有内容，本地与远程冲突

**解决**：
```bash
# 强制推送（慎用，会覆盖远程）
git push -u origin main --force
```
